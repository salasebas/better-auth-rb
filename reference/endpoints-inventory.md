## Notes

- **Body** lists JSON/form fields when OpenAPI metadata or `request_body_schema` helpers expose them. Endpoints with dynamic schemas may show `-`.
- **Query** and **Path params** include OpenAPI parameters and `:segments` from the route template.
- **Ruby API** maps each route to the in-process server helper on `auth.api`, using the same method names as `BetterAuth::API` (`auth.api.sign_up_email`, etc.).
- **Server-only** endpoints (metadata `SERVER_ONLY`) are intended for `auth.api`, not browser clients.
