# Grape Adapter Spec And Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `executing-plans` and `test-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a first-class Grape integration as `better_auth-grape`, plus `openauth-grape` alias.

**Architecture:** Keep authentication behavior in `packages/better_auth`; Grape stays a configuration, mounting, helper, SQL migration task, and documentation layer.

**Tech Stack:** Ruby, Grape 3.x, Rack, RSpec, rack-test.

---

## Public API And Interfaces

- `require "better_auth/grape"` and `require "better_auth_grape"`.
- `BetterAuth::Grape.configure`, `.auth`, `.reset!`, `.load_app_config`, `.migration_configuration`, and `.default_config_template`.
- `include BetterAuth::Grape` inside a `Grape::API` subclass to expose:
  - `helpers BetterAuth::Grape::Helpers`
  - `better_auth at: "/api/auth", auth: nil, **overrides`
  - `current_session`, `current_user`, `authenticated?`, and `require_authentication`.
- Rack mounted app wrapper that preserves Better Auth base path semantics when Grape/Rack splits `SCRIPT_NAME` and `PATH_INFO`.
- Rake tasks equivalent to Sinatra:
  - `better_auth:install`
  - `better_auth:generate:migration`
  - `better_auth:migrate`
  - `better_auth:routes`
- `packages/openauth-grape` alias gem requiring/loading `better_auth-grape`.

## Assumptions

- Use Grape v3 line: dependency `grape >= 3.0, < 4`.
- No Rails-style generator for v1; Grape has no universal app structure, so install/migration Rake tasks plus README examples are the right DX.
- External references checked during planning: Grape docs, Grape GitHub, and RubyGems grape.
