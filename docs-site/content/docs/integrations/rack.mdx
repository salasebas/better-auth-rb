description: Run RubyAuth as a plain Rack application or mount it in a custom stack.

<RubyAuthDisclaimer />

Use the core `better_auth` gem when you are not on Rails, Hanami, Sinatra, Roda, or Grape — or when you want full control over the Rack stack.

## Installation

```ruby title="Gemfile"
gem "better_auth"
gem "better_auth-cli"  # optional, for init/migrate/doctor
```

## Setup

<Steps>

### Scaffold (optional)

```bash
bundle exec better-auth init --cwd . --framework rack
```

This writes `config/better_auth.rb` and `db/better_auth/migrate/.keep`.

### Configure

```ruby title="config/better_auth.rb"
require "better_auth"

BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: :postgres,  # replace :memory before production
  email_and_password: { enabled: true }
)
```

### Run as the whole app

```ruby title="config.ru"
require_relative "config/better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: :postgres,
  email_and_password: { enabled: true }
)

run auth
```

### Mount under a prefix

```ruby title="config.ru"
require_relative "config/better_auth"

auth = BetterAuth.auth(/* ... */)

map "/api/auth" do
  run auth
end

map "/" do
  run MyMainApp
end
```

Set `base_path: "/api/auth"` in configuration when the public URL includes that prefix.

### Migrate

```bash
bundle exec better-auth generate \
  --cwd . \
  --config config/better_auth.rb \
  --dialect postgres \
  --output db/better_auth/schema.sql

bundle exec better-auth migrate --cwd . --config config/better_auth.rb --yes
bundle exec better-auth doctor --cwd . --config config/better_auth.rb
```

</Steps>

## Framework CLI init

`better-auth init --framework` supports `rails`, `hanami`, `sinatra`, `roda`, and `rack` — not Grape. Prefer the matching integration gem when one exists.

## Verify

```bash
curl http://localhost:9292/api/auth/ok
```
