Better Auth provides support for any social provider that implements the OAuth2 protocol or OpenID Connect (OIDC) flows through the [Generic OAuth Plugin](/docs/plugins/generic-oauth). You can use pre-configured helper functions for popular providers like Auth0, Keycloak, Okta, Microsoft Entra ID, and Slack, or manually configure any OAuth provider.

## Installation

<Steps>
  <Step>
    ### Add the plugin to your auth config

    To use the Generic OAuth plugin, add it to your auth config.

    ```ruby title="config/auth.rb"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
      # ... other config options
      plugins: [
        BetterAuth::Plugins.generic_oauth(
          config: [
            {
              provider_id: "provider-id",
              client_id: "test-client-id",
              client_secret: "test-client-secret",
              discovery_url: "https://auth.example.com/.well-known/openid-configuration"
              # ... other config options
            }
            # Add more providers as needed
          ]
        )
      ]
    )
    ```
  </Step>

  <Step>
    ### Start the OAuth flow from your Ruby app

    Use `auth.api.sign_in_with_oauth2` to generate the provider authorization URL. Pass `disable_redirect: true` when your app needs to return the URL instead of redirecting immediately.

    ```ruby title="sign-in.rb"
    response = auth.api.sign_in_with_oauth2(
      body: {
        provider_id: "provider-id",
        callback_url: "/dashboard",
        disable_redirect: true
      }
    )

    redirect_url = response.fetch(:url)
    ```
  </Step>
</Steps>

<Callout>
  Read more about installation and usage of the Generic Oauth plugin
  [here](/docs/plugins/generic-oauth#usage).
</Callout>

## Example Usage

Here's a basic example of configuring a generic OAuth provider:

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.generic_oauth(
      config: [
        {
          provider_id: "provider-id",
          client_id: ENV.fetch("CLIENT_ID"),
          client_secret: ENV.fetch("CLIENT_SECRET"),
          discovery_url: "https://auth.example.com/.well-known/openid-configuration"
        }
      ]
    )
  ]
)
```

## Using Pre-configured Providers

Better Auth provides pre-configured helper functions for popular OAuth providers. Here's an example using Slack:

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.generic_oauth(
      config: [
        BetterAuth::Plugins.slack(
          client_id: ENV.fetch("SLACK_CLIENT_ID"),
          client_secret: ENV.fetch("SLACK_CLIENT_SECRET")
        )
      ]
    )
  ]
)
```

```ruby title="sign-in.rb"
response = auth.api.sign_in_with_oauth2(
  body: {
    provider_id: "slack",
    callback_url: "/dashboard",
    disable_redirect: true
  }
)

redirect_url = response.fetch(:url)
```

For more pre-configured providers (Auth0, Keycloak, Okta, Microsoft Entra ID) and their configuration options, see the [Generic OAuth Plugin documentation](/docs/plugins/generic-oauth).

## Manual Configuration Example

If you need to configure a provider that doesn't have a pre-configured helper, make sure the config includes a way to fetch user profile data. Use OIDC discovery, a `user_info_url`, or a custom `get_user_info` callback. UserInfo JSON may use `sub` for the user ID; a custom `get_user_info` callback should map the provider response to `id`, `email`, and `name`.

### OAuth Provider With UserInfo Endpoint

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  # ... other config options
  plugins: [
    BetterAuth::Plugins.generic_oauth(
      config: [
        {
          provider_id: "custom-provider",
          client_id: ENV.fetch("CUSTOM_PROVIDER_CLIENT_ID"),
          client_secret: ENV.fetch("CUSTOM_PROVIDER_CLIENT_SECRET"),
          authorization_url: "https://auth.example.com/oauth/authorize",
          token_url: "https://auth.example.com/oauth/token",
          user_info_url: "https://api.example.com/oauth/userinfo",
          scopes: ["profile", "email"]
        }
      ]
    )
  ]
)
```

```ruby title="sign-in.rb"
response = auth.api.sign_in_with_oauth2(
  body: {
    provider_id: "custom-provider",
    callback_url: "/dashboard", # the path to redirect to after the user is authenticated
    disable_redirect: true
  }
)

redirect_url = response.fetch(:url)
```

The `user_info_url` endpoint should return JSON similar to this:

```json
{
  "sub": "user_123",
  "email": "user@example.com",
  "name": "Example User",
  "picture": "https://example.com/avatar.png",
  "email_verified": true
}
```

Providers with non-standard profile APIs, such as Instagram or Coinbase, usually need a provider-specific `get_user_info` callback to map their profile response to `id`, `email`, and `name`.
