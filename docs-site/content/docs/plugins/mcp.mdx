title: MCP

---

MCP OAuth behavior is provided by the OAuth Provider plugin.

<Callout type="info">
The standalone core `BetterAuth::Plugins.mcp` plugin was removed. Use `BetterAuth::Plugins.oauth_provider` from the `better_auth-oauth-provider` gem instead.
</Callout>

## Migration

1. Add the OAuth Provider gem:

```ruby
gem "better_auth-oauth-provider"
```

2. Require the plugin entrypoint and configure OAuth Provider:

```ruby title="config/auth.rb"
require "better_auth"
require "better_auth/oauth_provider"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "https://auth.example.com/api/auth"),
  plugins: [
    BetterAuth::Plugins.oauth_provider(
      scopes: ["openid", "profile", "email", "offline_access"],
      valid_audiences: ["https://api.example.com"]
    )
  ]
)
```

3. Protect MCP/resource-server routes with the OAuth Provider MCP helper:

```ruby
handler = BetterAuth::Plugins::OAuthProvider::MCP.mcp_handler(
  resource: "urn:api",
  resource_metadata_mappings: {
    "urn:api" => "https://api.example.com/.well-known/oauth-protected-resource"
  },
  verifier: ->(token) {
    BetterAuth::Plugins::OAuthProvider::ClientResource.verify_access_token(
      token,
      verify_options: {audience: "https://api.example.com", issuer: ENV.fetch("BETTER_AUTH_URL")},
      resource: "urn:api",
      resource_metadata_mappings: {"urn:api" => "https://api.example.com/.well-known/oauth-protected-resource"}
    )
  }
) { |_request, claims| claims }
```

See the [OAuth Provider](/docs/plugins/oauth-provider) guide for the full OAuth 2.1, OIDC, dynamic registration, introspection, and resource-server surface.
