<RubyAuthDisclaimer />

The One-Time Token (OTT) plugin provides functionality to generate and verify secure, single-use session tokens. These are commonly used for across domains authentication.

## Installation

<Steps>
  <Step>
    ### Add the plugin to your auth config

    To use the One-Time Token plugin, add it to your auth config.

    ```ruby title="config/auth.rb"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
      plugins: [
        BetterAuth::Plugins.one_time_token
      ]
    )
    ```
  </Step>

</Steps>

## Usage

### 1. Generate a Token


<APIMethod path="/one-time-token/generate" method="GET" requireSession>
  ```
  type generateOneTimeToken = {
  }
  ```
</APIMethod>

This will return a `token` that is attached to the current session which can be used to verify the one-time token. By default, the token will expire in 3 minutes.

### 2. Verify the Token


<APIMethod path="/one-time-token/verify" method="POST">
  ```
  type verifyOneTimeToken = {
      /**
       * The one-time token.
       */
      token: string = "one-time-token"
  }
  ```
</APIMethod>

This will return the session that was attached to the token.

## Options

These options can be configured when adding the `one_time_token` plugin:

* **`disable_client_request`** (`Boolean`): Optional. If `true`, the token will only be generated on the server side. Default: `false`.
* **`expires_in`** (`Integer`): Optional. The duration for which the token is valid in minutes. Default: `3`.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.one_time_token(
      disable_client_request: true,
      expires_in: 5
    )
  ]
)
```

* **`generate_token`**: A custom token generator function that takes the session object and `ctx` as parameters.

* **`store_token`**: Optional. This option allows you to configure how the token is stored in your database.

  * **`plain`**: The token is stored in plain text. (Default)
  * **`hashed`**: The token is hashed using the default hasher.
  * **`custom-hasher`**: A custom hasher function that takes a token and returns a hashed token.

<Callout type="info">
  Note: It will not affect the token that's sent, it will only affect the token stored in your database.
</Callout>

Examples:

```ruby title="server.rb"
generated = auth.api.generate_one_time_token(
  headers: {"cookie" => session_cookie}
)

generated[:token]
```

```ruby title="server.rb"
restored = auth.api.verify_one_time_token(
  body: {token: generated[:token]}
)

restored[:user]
```

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  plugins: [
    BetterAuth::Plugins.one_time_token(
      store_token: "hashed",
      generate_token: ->(_session, _ctx) { SecureRandom.hex(24) }
    )
  ]
)
```
