<RubyAuthDisclaimer />

RubyAuth uses Better Auth `v1.6.23` as its current source reference. The Ruby port implements a large server-side surface, but exact upstream parity is still being tightened across some routes, adapter edge cases, OpenAPI schemas, and docs pages.

Use this page as a conservative support inventory. A feature listed as supported means RubyAuth has Ruby server-side documentation or package coverage for it. Browser client packages, JavaScript framework integrations, and upstream-only helpers are outside the Ruby server scope unless a Ruby package explicitly documents an equivalent.

## Intentionally Omitted or Unsupported Upstream Docs

| Area | RubyAuth status |
| --- | --- |
| Browser client packages | Not documented as supported Ruby server features. Call RubyAuth HTTP endpoints from your frontend, mobile client, HTMX, Turbo, or server-rendered forms. |
| JavaScript framework integrations | Not documented as supported Ruby integrations. Use the Ruby framework integrations listed above. |
| `mcp` | Not documented as a supported RubyAuth plugin. |
| Upstream `oidc-provider` | Not documented as a supported RubyAuth plugin. RubyAuth documents its own [OAuth Provider](/docs/plugins/oauth-provider) package separately. |
| Non-Stripe payment plugins | Not documented as supported RubyAuth payment plugins. |
| AI and infrastructure pages | Not documented as supported RubyAuth features unless a Ruby package or docs page explicitly covers the feature. |
