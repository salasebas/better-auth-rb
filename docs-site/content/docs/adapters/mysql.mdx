MySQL is a popular open-source relational database management system (RDBMS) that is widely used for building web applications and other types of software. It provides a flexible and scalable database solution that allows for efficient storage and retrieval of data.
Read more here: [MySQL](https://www.mysql.com/).

<Callout type="info">
Use the lambda form when configuring the direct adapter. Better Auth passes the
final configuration into the lambda, so the adapter sees plugins, custom model
names, custom field names, and `advanced.database.generate_id`.
</Callout>

## Install Dependencies

```ruby title="Gemfile"
gem "better_auth"
gem "mysql2"
```

```bash title="Terminal"
bundle install
```

## Example Usage

Make sure you have MySQL installed and configured.
Then, you can connect it straight into Better Auth.

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: ->(options) {
    BetterAuth::Adapters::MySQL.new(
      options,
      url: ENV.fetch("DATABASE_URL")
    )
  }
)
```

<Callout>
  For more information, read the Ruby
  `BetterAuth::Adapters::MySQL` implementation and adapter tests in this repository.
</Callout>

You may also pass an existing `Mysql2::Client` when your app owns connection
management.

```ruby title="config/auth.rb"
require "mysql2"

client = Mysql2::Client.new(
  host: "localhost",
  username: "root",
  password: ENV.fetch("MYSQL_PASSWORD"),
  database: "app_development",
  symbolize_keys: false
)

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  database: ->(options) {
    BetterAuth::Adapters::MySQL.new(
      options,
      connection: client
    )
  }
)
```

## Rails

Rails apps should normally use `better_auth-rails` and the ActiveRecord adapter.

```ruby title="Gemfile"
gem "better_auth-rails"
gem "mysql2"
```

```bash title="Terminal"
bin/rails generate better_auth:install
bin/rails db:migrate
```

## Joins (Experimental)

Database joins is useful when Better-Auth needs to fetch related data from multiple tables in a single query.
Endpoints like `/get-session`, `/get-full-organization` and many others benefit greatly from this feature,
seeing upwards of 2x to 3x performance improvements depending on database latency.

The Ruby MySQL adapter supports joins.

To enable this feature, you need to set the `experimental.joins` option to `true` in your auth configuration.

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  experimental: { joins: true },
  database: ->(options) {
    BetterAuth::Adapters::MySQL.new(
      options,
      url: ENV.fetch("DATABASE_URL")
    )
  }
)
```

<Callout type="warn">
  It's possible that you may need to run migrations after enabling this feature.
</Callout>

## Schema

The Ruby schema maps Better Auth logical fields to MySQL physical columns.

| Better Auth field | MySQL column |
| --- | --- |
| `emailVerified` | `email_verified` |
| `userId` | `user_id` |
| `createdAt` | `created_at` |
| `expiresAt` | `expires_at` |

MySQL DDL uses InnoDB, `utf8mb4`, `json` for JSON-like fields, `datetime(6)`
for date fields, and `varchar(191)` for indexed strings. Ids are generated as
hex strings by default. Set
`advanced: { database: { generate_id: "uuid" } }` to generate UUID strings
instead.

## Additional Information

MySQL is supported under the hood via `BetterAuth::Adapters::MySQL`. Other relational database options are documented in <Link href="/docs/adapters/other-relational-databases">Other Relational Databases</Link>.

If you're looking for performance improvements or tips, take a look at our guide to <Link href="/docs/guides/optimizing-for-performance">performance optimizations</Link>.

- The direct adapter supports the shared Better Auth CRUD contract, including
  operators, sorting, pagination, counts, transactions, and native joins.
- If you pass your own `Mysql2::Client`, configure connection settings such as
  timezone/session behavior in that client before handing it to Better Auth.
- Rails apps should prefer Rails migrations generated by `better_auth-rails`;
  direct SQL generation is intended for non-Rails apps or custom migration
  workflows.
