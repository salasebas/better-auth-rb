title: unable_to_create_user
description: The user could not be created during authentication.
---

## What is it?

This error occurs when RubyAuth fails to create a new user during the authentication process. It typically happens during OAuth or SSO-based signup flows when a new user record is expected to be created but the operation fails.

## Common Causes

* Database write failure due to connection issues, timeouts, or transaction errors.
* Missing or invalid required fields in the user schema.
* Unique constraint violations (e.g., email already exists).
* Mismatch between your database schema and the expected RubyAuth schema.
* Errors thrown inside custom database hooks (e.g., `user.create`).
* Misconfigured adapters or database clients.

## How to resolve

### Verify database connectivity

* Ensure your database is reachable and properly configured.
* Check for connection pool issues, timeouts, or failed queries.

### Validate schema and constraints

* Make sure all required user fields are present and correctly typed.
* Check for unique constraints (e.g., email conflicts).

### Run migrations

* Ensure your database schema is up to date with the current RubyAuth version.

### Review custom hooks

* If you are using `database_hooks.user.create`, ensure it is not throwing errors or returning invalid data.

### Inspect logs

* Check server logs for detailed error messages during user creation.

## When it occurs

The request reached RubyAuth but failed validation for this specific callback or auth flow.

## HTTP status

RubyAuth normally returns `400 Bad Request` for this error, unless the route redirects to a configured error callback URL.

## Example response

```json
{
  "code": "UNABLE_TO_CREATE_USER",
  "message": "The request reached RubyAuth but failed validation for this specific callback or auth flow."
}
```

## How to fix

Verify the RubyAuth route, provider settings, callback URL, cookies, and `BETTER_AUTH_SECRET` used by every running instance. In Rails, also confirm the mounted auth path matches `config.base_path`; in Rack, confirm requests reach the same `auth` instance that initiated the flow.
