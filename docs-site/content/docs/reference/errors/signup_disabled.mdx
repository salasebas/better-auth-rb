---
title: signup_disabled
description: Signup disabled error
---

This error occurs when you disable sign up in your oauth provider config and a user tries to sign up with that provider.

## How to fix

If you're using the `disableSignUp` option with stateless mode, you will see this error. Please consider using database hooks instead to handle this case.

```ruby title="config/initializers/better_auth.rb"
curl -i http://localhost:3000/api/auth/example

# JSON response
# { "code": "ERROR_CODE", "message": "Human readable message" }
```

## When it occurs

The sign-up endpoint was called while email/password sign-up is disabled for this server configuration.

## HTTP status

RubyAuth normally returns `400 Bad Request` for this error, unless the route redirects to a configured error callback URL.

## Example response

```json
{
  "code": "SIGNUP_DISABLED",
  "message": "The sign-up endpoint was called while email/password sign-up is disabled for this server configuration."
}
```

## How to fix

Verify the RubyAuth route, provider settings, callback URL, cookies, and `BETTER_AUTH_SECRET` used by every running instance. In Rails, also confirm the mounted auth path matches `config.base_path`; in Rack, confirm requests reach the same `auth` instance that initiated the flow.
