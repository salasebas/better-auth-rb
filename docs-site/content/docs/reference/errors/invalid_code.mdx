---
title: invalid_code
description: The provided authentication code is invalid or expired.
---

This error occurs when the authentication code is invalid, expired, or cannot be verified.

## How to fix

* Restart the authentication flow to generate a new code
* Ensure the code is used only once and within its valid time window
* Verify that the correct code is being sent to the callback endpoint

## When it occurs

The callback or verification route expected a one-time code, but the supplied code was missing, expired, already consumed, or did not match stored verification state.

## HTTP status

RubyAuth normally returns `400 Bad Request` for this error, unless the route redirects to a configured error callback URL.

## Example response

```json
{
  "code": "INVALID_CODE",
  "message": "The callback or verification route expected a one-time code, but the supplied code was missing, expired, already consumed, or did not match stored verification state."
}
```

## How to fix

Verify the RubyAuth route, provider settings, callback URL, cookies, and `BETTER_AUTH_SECRET` used by every running instance. In Rails, also confirm the mounted auth path matches `config.base_path`; in Rack, confirm requests reach the same `auth` instance that initiated the flow.
