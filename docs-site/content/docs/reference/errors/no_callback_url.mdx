---
title: no_callback_url
description: The callback URL was not found in the request.
---

## What is it?

This error occurs during the OAuth flow when the request reaches your `/api/auth/callback` endpoint
but the `state` data does not contain a callback URL.
RubyAuth stores metadata in `state` when the flow starts, including where to redirect after a
successful sign-in/link. If that URL is missing at callback time, we cannot safely continue.

## Common Causes

* The OAuth flow was not started via RubyAuth APIs, so the `state` payload never included a callback URL.
* A reverse proxy, CDN, or middleware altered the flow, causing the app to read a different or empty `state`.

## How to resolve

### Start the flow through RubyAuth

* Always initiate OAuth using RubyAuth's built-in methods so `state` is generated with the needed fields.

## When it occurs

A callback URL was required for the flow but was absent from the request or provider configuration.

## HTTP status

RubyAuth normally returns `400 Bad Request` for this error, unless the route redirects to a configured error callback URL.

## Example response

```json
{
  "code": "NO_CALLBACK_URL",
  "message": "A callback URL was required for the flow but was absent from the request or provider configuration."
}
```

## How to fix

Verify the RubyAuth route, provider settings, callback URL, cookies, and `BETTER_AUTH_SECRET` used by every running instance. In Rails, also confirm the mounted auth path matches `config.base_path`; in Rack, confirm requests reach the same `auth` instance that initiated the flow.
