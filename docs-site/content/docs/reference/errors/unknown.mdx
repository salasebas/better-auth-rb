If you're seeing this page, it means that an unknown error occurred.
Please try again or return to the home page.

If you believe this error comes from RubyAuth core functionality or a plugin,
please open an issue on [GitHub](https://github.com/better-auth/better-auth/issues).

To view all known errors, please visit the [Error Codes Reference](/docs/reference/errors) page.

## When it occurs

The provider or route returned an error that did not map to a more specific RubyAuth error code.

## HTTP status

RubyAuth normally returns `500 Internal Server Error` for this error, unless the route redirects to a configured error callback URL.

## Example response

```json
{
  "code": "UNKNOWN",
  "message": "The provider or route returned an error that did not map to a more specific RubyAuth error code."
}
```

## How to fix

Verify the RubyAuth route, provider settings, callback URL, cookies, and `BETTER_AUTH_SECRET` used by every running instance. In Rails, also confirm the mounted auth path matches `config.base_path`; in Rack, confirm requests reach the same `auth` instance that initiated the flow.
