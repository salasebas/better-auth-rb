---

## When this appears

The OAuth state cookie does not match the `state` query parameter on callback.

Redirect URL shape:

```
/api/auth/error?error=state_mismatch
```

Or your configured `errorCallbackURL` with the same query parameters.

## Common causes

- Expired or missing state cookie
- User opened callback in a different browser
- Cookie blocked or wrong domain

## How to fix

- Ensure cookies work on your auth domain
- Do not open OAuth callbacks in private windows without cookies
- Check `trusted_origins` includes your frontend URL

## Diagnostics

```bash
bundle exec better-auth doctor --cwd . --config config/better_auth.rb
```
