title: Migrating from Clerk to RubyAuth
description: A step-by-step guide to transitioning from Clerk to RubyAuth.
---

<RubyAuthDisclaimer />

In this guide, we'll walk through the steps to migrate a project from Clerk to RubyAuth — including email/password with proper hashing, social/external accounts, phone number, two-factor data, and more.

<Callout type="warn">
  This migration will invalidate all active sessions. This guide doesn't currently show you how to migrate Organization but it should be possible with additional steps and the [Organization](/docs/plugins/organization) Plugin.
</Callout>

## Before You Begin

Before starting the migration process, set up RubyAuth in your project. Follow the [installation guide](/docs/installation) to get started. And go to

<Steps>
  <Step>
    ### Connect to your database

    You'll need to connect to your database to migrate the users and accounts. You can use any database you want, but for this example, we'll use PostgreSQL.

    ```ruby title="Gemfile"
    gem "pg"

    ```

    And then you can use the following code to connect to your database.

    ```ruby title="config/initializers/better_auth.rb"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
      database: :postgres,
      email_and_password: { enabled: true },
      social_providers: {
        google: {
          client_id: ENV.fetch("GOOGLE_CLIENT_ID"),
          client_secret: ENV.fetch("GOOGLE_CLIENT_SECRET")
        }
      }
    )
    ```
  </Step>

  <Step>
    ### Enable Email and Password (Optional)

    Enable the email and password in your auth config and implement your own logic for sending verification emails, reset password emails, etc.

    <Callout type="info">
      **Important for Clerk Migrations**: Clerk uses bcrypt to hash passwords, while RubyAuth uses `scrypt` by default. To ensure migrated users can sign in with their existing passwords, you'll need to configure RubyAuth to use bcrypt for password verification.
    </Callout>

    First, install bcrypt:

    ```ruby title="Gemfile"
    bundle add bcrypt

    ```

    Then configure RubyAuth to use bcrypt:

    ```ruby title="config/initializers/better_auth.rb"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
      database: :postgres,
      email_and_password: { enabled: true },
      social_providers: {
        google: {
          client_id: ENV.fetch("GOOGLE_CLIENT_ID"),
          client_secret: ENV.fetch("GOOGLE_CLIENT_SECRET")
        }
      }
    )
    ```

    See [Email and Password](/docs/authentication/email-password) for more configuration options.
  </Step>

  <Step>
    ### Setup Social Providers (Optional)

    Add social providers you have enabled in your Clerk project in your auth config.

    ```ruby title="config/initializers/better_auth.rb"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
      database: :postgres,
      email_and_password: { enabled: true },
      social_providers: {
        google: {
          client_id: ENV.fetch("GOOGLE_CLIENT_ID"),
          client_secret: ENV.fetch("GOOGLE_CLIENT_SECRET")
        }
      }
    )
    ```
  </Step>

  <Step>
    ### Add Plugins (Optional)

    You can add the following plugins to your auth config based on your needs.

    [Admin](/docs/plugins/admin) Plugin will allow you to manage users, user impersonations and app level roles and permissions.

    [Two Factor](/docs/plugins/2fa) Plugin will allow you to add two-factor authentication to your application.

    [Phone Number](/docs/plugins/phone-number) Plugin will allow you to add phone number authentication to your application.

    [Username](/docs/plugins/username) Plugin will allow you to add username authentication to your application.

    ```ruby title="config/initializers/better_auth.rb"
    # scripts/migrate_clerk.rb
    require "csv"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL"),
      database: :postgres,
      email_and_password: { enabled: true }
    )

    CSV.foreach("exports/clerk_users.csv", headers: true) do |row|
      user = {
        email: row.fetch("email"),
        name: row["name"],
        email_verified: row["email_verified"] == "true"
      }

      # Prefer admin APIs when enabled; otherwise insert through your audited app import service.
      auth.api.create_user(user: user) if auth.api.respond_to?(:create_user)
    end

    # Existing clerk sessions cannot be reused by RubyAuth. Plan a session invalidation window.
    ```
  </Step>

  <Step>
    ### Generate Schema

    If you're using a custom database adapter, generate the schema:

    ```sh
    bundle exec better-auth generate --cwd . --dialect postgres --output db/better_auth/schema.sql --config config/better_auth.rb
    ```

    or if you're using the default adapter, you can use the following command:

    ```sh
    bundle exec better-auth migrate --cwd . --config config/better_auth.rb --yes
    ```
  </Step>

  <Step>
    ### Export Clerk Users

    Go to the Clerk dashboard and export the users. Check how to do it [here](https://clerk.com/docs/deployments/exporting-users#export-your-users-data-from-the-clerk-dashboard). It will download a CSV file with the users data. You need to save it as `exported_users.csv` and put it in the root of your project.
  </Step>

  <Step>
    ### Create the migration script

    Create a new file called `migrate_clerk.rb` in the `scripts` folder and add the following code:

    ```ruby title="scripts/migrate-clerk.rb"
    # scripts/migrate_clerk.rb
    require "csv"
    require "better_auth"

    auth = BetterAuth.auth(
      secret: ENV.fetch("BETTER_AUTH_SECRET"),
      base_url: ENV.fetch("BETTER_AUTH_URL"),
      database: :postgres,
      email_and_password: { enabled: true }
    )

    CSV.foreach("exports/clerk_users.csv", headers: true) do |row|
      user = {
        email: row.fetch("email"),
        name: row["name"],
        email_verified: row["email_verified"] == "true"
      }

      # Prefer admin APIs when enabled; otherwise insert through your audited app import service.
      auth.api.create_user(user: user) if auth.api.respond_to?(:create_user)
    end

    # Existing clerk sessions cannot be reused by RubyAuth. Plan a session invalidation window.
    ```

    Make sure to replace the `process.env.CLERK_SECRET_KEY` with your own Clerk secret key. Feel free to customize the script to your needs.
  </Step>

  <Step>
    ### Run the migration

    Run the migration:

    ```sh
    bundle exec ruby scripts/migrate_clerk.rb
    ```

    <Callout type="warning">
      Make sure to:

      1. Test the migration in a development environment first
      2. Monitor the migration process for any errors
      3. Verify the migrated data in RubyAuth before proceeding
      4. Keep Clerk installed and configured until the migration is complete
    </Callout>
  </Step>

  <Step>
    ### Verify the migration

    After running the migration, verify that all users have been properly migrated by checking the database.
  </Step>

  <Step>
    ### Update your components

    Now that the data is migrated, you can start updating your components to use RubyAuth. Here's an example for the sign-in component:

    ```ruby title="components/auth/sign-in.rb"
    # Rails controller/helper example
    before_action :require_current_user

    def require_current_user
      @session = BetterAuth::Rails.current_session(request)
      redirect_to "/sign-in" unless @session
    end
    ```
  </Step>

  <Step>
    ### Update the middleware

    Replace your Clerk middleware with RubyAuth's middleware:

    ```ruby title="app/controllers/application_controller.rb"
    # Rails controller/helper example
    before_action :require_current_user

    def require_current_user
      @session = BetterAuth::Rails.current_session(request)
      redirect_to "/sign-in" unless @session
    end
    ```
  </Step>

  <Step>
    ### Remove Clerk Dependencies

    Once you've verified that everything is working correctly with RubyAuth, you can remove Clerk:

    ```bash
    # Remove the old provider SDK or gem after RubyAuth traffic is cut over.
    ```
  </Step>
</Steps>
