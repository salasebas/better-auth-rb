In this guide, we’ll walk you through the steps of creating your first RubyAuth plugin.

<Callout type="warn">
  This guide assumes you have <Link href="/docs/installation">setup the basics</Link> of RubyAuth and are ready to create your first plugin.
</Callout>

<Steps>
  <Step>
    ## Plan your idea

    Before beginning, you must know what plugin you intend to create.

    In this guide, we’ll create a **birthday plugin** to keep track of user birth dates.
  </Step>

  <Step>
    ## Server plugin first

    RubyAuth plugins are server-side `BetterAuth::Plugin` objects. The server plugin forms the foundation of your authentication system, while Rails, Rack, or browser HTTP code can call the endpoints it exposes.

    <Callout>
      You can read more about server plugins in our <Link href="/docs/concepts/plugins#creating-a-plugin">documentation</Link>.
    </Callout>

    ### Creating the server plugin

    Go ahead and find a suitable location to create your birthday plugin folder, with an `index.rb` file within.

    <Files>
      <Folder name="birthday-plugin" defaultOpen>
        <File name="index.rb" />
      </Folder>
    </Files>

    In the `index.rb` file, we’ll export a function that represents our server plugin.
    This will be what we will later add to our plugin list in the `config/initializers/better_auth.rb` file.

    ```ruby title="index.rb"
module BetterAuth
  module Plugins
    module_function

    def hello_world(**options)
      BetterAuth::Plugin.new(
        id: "hello-world",
        endpoints: {
          hello_world: BetterAuth::Endpoint.new(path: "/hello-world", method: "GET") do |ctx|
            ctx.json({ message: options.fetch(:message, "Hello World") })
          end
        }
      )
    end
  end
end
```

    Although this does nothing, you have technically just made yourself your first plugin, congratulations! 🎉
  </Step>

  <Step>
    ### Defining a schema

    In order to save each user’s birthday data, we must create a schema on top of the `user` model.

    By creating a schema here, this also allows <Link href="/docs/concepts/cli">RubyAuth’s CLI</Link> to generate the schemas required to update your database.

    <Callout type="info">
      You can learn more about <Link href="/docs/concepts/plugins#schema">plugin schemas here</Link>.
    </Callout>

    ```ruby title="index.rb"
module BetterAuth
  module Plugins
    module_function

    def hello_world(**options)
      BetterAuth::Plugin.new(
        id: "hello-world",
        endpoints: {
          hello_world: BetterAuth::Endpoint.new(path: "/hello-world", method: "GET") do |ctx|
            ctx.json({ message: options.fetch(:message, "Hello World") })
          end
        }
      )
    end
  end
end
```
  </Step>

  <Step>
    ### Authorization logic

    For this example guide, we’ll set up authentication logic to check and ensure that the user who signs-up is older than 5.
    But the same concept could be applied for something like verifying users agreeing to the TOS or anything alike.

    To do this, we’ll utilize <Link href="/docs/concepts/plugins#hooks">Hooks</Link>, which allows us to run code `before` or `after` an action is performed.

    ```ruby title="index.rb"
module BetterAuth
  module Plugins
    module_function

    def hello_world(**options)
      BetterAuth::Plugin.new(
        id: "hello-world",
        endpoints: {
          hello_world: BetterAuth::Endpoint.new(path: "/hello-world", method: "GET") do |ctx|
            ctx.json({ message: options.fetch(:message, "Hello World") })
          end
        }
      )
    end
  end
end
```

    In our case we want to match any requests going to the signup path:

    ```ruby title="Before hook"
BetterAuth::Plugin.new(
  id: "hello-world",
  middlewares: [
    {
      path: "/hello-world",
      middleware: ->(ctx) { ctx.headers["x-plugin"] = "hello"; nil }
    }
  ]
)
```

    And for our logic, we’ll write the following code to check the if user’s birthday makes them above 5 years old.

    ```ruby title="Imports"
module BetterAuth
  module Plugins
    module_function

    def hello_world(**options)
      BetterAuth::Plugin.new(
        id: "hello-world",
        endpoints: {
          hello_world: BetterAuth::Endpoint.new(path: "/hello-world", method: "GET") do |ctx|
            ctx.json({ message: options.fetch(:message, "Hello World") })
          end
        }
      )
    end
  end
end
```

    ```ruby title="Before hook"
BetterAuth::Plugin.new(
  id: "hello-world",
  middlewares: [
    {
      path: "/hello-world",
      middleware: ->(ctx) { ctx.headers["x-plugin"] = "hello"; nil }
    }
  ]
)
```

    **Authorized!** 🔒

    We’ve now successfully written code to ensure authorization for users above 5!
  </Step>

  <Step>
    ## Client Boundary

    We’re close to the finish line! 🏁

    Now that we have created our server plugin, the next step is to decide how application code calls it.
    RubyAuth does not ship a JavaScript client plugin contract; call the endpoint from Rails, Rack, or browser HTTP code.

    First, let’s create a small HTTP wrapper:

    <Files>
      <Folder name="birthday-plugin" defaultOpen>
        <File name="index.rb" />

        <File name="client.rb" />
      </Folder>
    </Files>

    Then, add the following code:

    ```ruby title="client.rb"
# Client plugins are not part of RubyAuth's server-side plugin contract.
# Expose server endpoints and call them with HTTP from your Rails/Rack app.
```

    What we’ve done is keep the Ruby plugin contract server-side while still giving the application a small, testable call site.
  </Step>

  <Step>
    ## Initiate your plugin!


    ### Server initiation

    ```ruby title="config.ru"
module BetterAuth
  module Plugins
    module_function

    def hello_world(**options)
      BetterAuth::Plugin.new(
        id: "hello-world",
        endpoints: {
          hello_world: BetterAuth::Endpoint.new(path: "/hello-world", method: "GET") do |ctx|
            ctx.json({ message: options.fetch(:message, "Hello World") })
          end
        }
      )
    end
  end
end
```

    ### Client boundary

    ```ruby title="app/services/birthday_plugin_client.rb"
    require "net/http"
    require "json"

    class BirthdayPluginClient
      def initialize(base_url:)
        @base_url = URI(base_url)
      end

      def hello_world
        uri = @base_url + "/api/auth/hello-world"
        JSON.parse(Net::HTTP.get(uri))
      end
    end
    ```

    ### Oh yeah, the schemas!

    Don’t forget to add your `birthday` field to your `user` table model!

    Or, use the `generate` <Link href="/docs/concepts/cli#generate">CLI command</Link>:

    ```bash
    bundle exec better-auth generate --cwd . --dialect postgres --output db/better_auth/schema.sql --config config/better_auth.rb
    ```
  </Step>
</Steps>
