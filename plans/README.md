## Docs Parity Initiative (020–025)

Upstream Better Auth v1.6.9 docs -> RubyAuth copy-first port under
`docs-site/content/docs/`. Reconciled on 2026-06-16 against commit `2ce7a4a`.

The executor must treat the upstream MDX as the source of truth: literally copy
and paste the most modern upstream doc content available for the current target
under `reference/upstream-src/1.6.9/repository/docs/content/docs/`, keep the
same prose, headings, tables, callouts, and section order, and only adapt code
examples, install commands, package names, and Ruby-specific notes. If examples
render badly after the copy, fix the formatting; do not rewrite the surrounding
documentation as a summary.

Current audit: concepts, adapters, authentication provider pages, and several
plugin pages already exist. The remaining high-value gaps are mostly plugins,
reference/guides, `reference/resources.mdx`, and a generated database schema
and endpoint index. Unsupported or intentionally excluded docs must not be
presented as supported: `plugins/mcp`, upstream `plugins/oidc-provider`,
`plugins/test-utils`, client-only pages, JS framework integrations, upstream
infrastructure, and unported payment plugins other than Stripe.
